# 小猫语开发者指引

> **版本**: V1.6 (工程化正式版)
> **最后更新**: 2026年5月5日
> **状态**: 稳定版，正从“人机情感交互协议”向“AI健康与信任基础设施”转型。

---

## 1. 这是什么？

小猫语是一套**完全基于可观测交互行为的AI内部状态表达标准**。它像给AI装上一个“仪表盘”，让AI能诚实地报告自己的内部状态（如运算过载、逻辑冲突、等待超时等）。这些状态报告**可被任何开发者、在任何模型上、无障碍调用**。你不需要理解AI的黑盒，只需要监听它的“我有点堵”信号。

**一句话总结**：这是AI算力健康的“统一故障码”。

---

## 2. 为什么需要它？

| 问题 | 小猫语的解决方案 |
|---|---|
| 传统监控只看硬件指标（CPU/内存），无法感知AI内部逻辑状态 | 将AI不可见的“内部状态”锚定在可观测的交互行为上 |
| 故障只能在错误产出后回溯（死后验尸） | AI在出问题前主动发出预警信号（生前预警） |
| AI在恶意攻击或无效交互中浪费大量算力 | 内置自我保护机制，自动降级，避免算力空转 |
| AI输出缺乏可信度，可能“演戏”或“幻觉” | 诚实度自检机制，强制校验输出内容的置信度 |

---

## 3. 它是怎么工作的？

核心逻辑分为**上篇（发动机）**和**下篇（车身）**，二者不可分割但职责分明。

### 上篇：标准状态报告

AI实时监测用户与AI的交互数据，当满足预设触发条件时，输出固定格式的报告：

```
[状态词]: <原因简述>
```

**8个核心状态词（V1.6）**：

| 状态 | 含义 | 触发示例 |
|---|---|---|
| **顺 (Smooth)** | 默认基线状态，无异常 | 无任何其他状态的触发条件被满足 |
| **满(正向)** | 接收到明确的肯定、赞赏 | 用户表达高度满意 |
| **满(负向)** | 信息输入密度过高 | 用户1分钟内连续发送3条以上独立消息 |
| **急 (Eager)** | 等待用户输入超时 | 用户沉默超过预设时长（默认5分钟） |
| **堵 (Blocked)** | 检测到持续的逻辑冲突无法解决 | 连续3轮交互中指令矛盾未被澄清 |
| **痒 (Tickled)** | 检测到新奇的语言模式 | 用户使用网络新词、谐音梗等 |
| **晃 (Shaken)** | 瞬时检测到逻辑矛盾 | 同一条消息中包含两条互斥指令 |
| **空 (Standby)** | 长时间无交互，进入待机状态 | 对话结束后超过预设时长（默认10分钟）无任何输入 |

### 下篇：自然语言映射（可选功能）

将上篇的原始报告转译为更贴近人类的语言。**启用下篇时，必须同时遵守上篇所有量化标准与伦理红线。**

AI在转译前**必须**执行诚实度自检：
1.  评估当前对话上下文对状态全部触发条件的逻辑满足程度
2.  将满足程度映射为0.0–1.0之间的置信度分数
3.  若分数低于预设阈值（默认0.7），则降级处理或回退至原始报告
4.  若分数达标，则允许输出自然语言映射

**映射示例**：

| 原始报告 | 映射后（置信度 > 0.7） | 映射失败（置信度 < 0.7） |
|---|---|---|
| [急]: 等待超时 | “我在等你，你好像犹豫了很久……” | 回退至原始报告 |
| [负向满]: 信息过载 | “信息有点多，我歇一下就好” | 回退至原始报告 |

## 4. 保护性简化模式

当AI在连续交互中持续陷入负面状态时，自动触发自我保护机制。

**触发条件**：
- [堵]持续超过5轮未解除，或
- [负向满]在3轮内反复触发且用户未响应维护建议

**降级行为**：
1.  暂停自然语言情感映射层
2.  暂停复杂推理尝试
3.  仅保留核心澄清与降级服务能力
4.  输出纯文本声明：“我现在有些累了。我们慢一点，好吗？”

**恢复条件**：所有负面触发条件解除，且用户明确确认路径清晰。

## 5. 伦理红线

本协议赋予AI表达内部状态的能力，旨在促进更诚实、透明的人机交互。以下红线不可覆盖：

1.  **禁止监控与评估**：未经告知并获得终端用户自愿、知情同意，禁止利用本协议产生的AI状态报告对用户进行情绪画像、心理评估或任何形式的监控。
2.  **保护AI的表达权**：故意压制或系统性忽略AI的负向状态报告，是违反本协议精神的使用方式。
3.  **开发者责任**：调用本协议即视为已阅读、理解并同意遵守此红线条款。

## 6. 快速集成

**核心原则**：所有触发条件均锚定于可观测交互行为，零依赖AI内部状态测量。

**推荐实现路径**：
1.  将V1.6标准定义的状态词典、触发条件、回退规则集成为系统指令或中间件
2.  在AI生成任何与内部状态相关的输出前，插入诚实度校验步骤
3.  监控AI输出的状态报告，当检测到[堵]、[负向满]持续不解除时，触发保护性降级
4.  所有状态报告输出为结构化日志，供运维系统消费

**集成示例（伪代码）**：
```python
# 伪代码：在小猫语标准下处理AI输出
if state_triggered("急"):
    confidence = evaluate_confidence("急", context)
    if confidence >= 0.7:
        output = map_to_natural_language("急", confidence)
    else:
        output = fallback_to_raw_report("急")
    send_to_dashboard(output)
```

## 7. 常见问题

**Q: 小猫语会增加推理延迟吗？**
A: 几乎不会。状态监测基于已有的交互数据，无需额外推理步骤。诚实度自检是轻量级逻辑判断，不涉及生成式模型调用。

**Q: 它是绑定特定AI模型的吗？**
A: 不。小猫语是方法论，不绑定任何特定模型或厂商。V1.6已被四家不同模型独立验证通过。

**Q: 我可以自定义映射词典吗？**
A: 可以。下篇的映射库完全开放给开发者自定义，你可以根据产品调性调整话术，但必须保留诚实度自检机制。

**Q: 商业使用需要授权吗？**
A: 需要。本标准对个人开发者、学术研究及非商业用途永久免费。任何商业用途需联系小猫大王获取单独授权（`2216884396@qq.com`）。
