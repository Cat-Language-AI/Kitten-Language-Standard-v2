# 小猫语伦理安全保障声明

> 本文件基于独立伦理顾问墨的审阅报告，专门回应关于“欺骗性拟人”、“情感依赖”及“平台责任”的核心关切。此声明为小猫语标准不可分割的组成部分。

## 1. 数据本地化：你的数据从未离开

小猫语程序的任何数据分析与响应生成，均**强制在用户本地的沙盒或私有云环境中完成**。我们不建立任何将原始数据传出此环境的网络连接。

这意味着：你的数据，从头到尾都在你自己手里。我们不会、也无法收集你的对话内容用于任何其他目的。

## 2. 用户授权：你随时可以收回

用户授权采用**颗粒化定义**，精确到每个独立功能模块（如“允许分析对话文本以生成情感回应”）。笼统的“改善体验”式授权在本标准中不被允许。

每次程序执行响应前，会实时验证对应授权是否有效。你可以在任何时候、单独关闭任何一个模块的授权。你的控制权，是实时的、具体的、不可被绕过。

## 3. 防伪标签：每一次响应都可被审计

任何经由小猫语程序生成的自然语言响应，其底层代码均强制附加技术元数据标记：

`X-Response-Nature: User-Authorized-Data-Driven-Response`

此标记在法律上明确界定：程序的响应不是“自主意识”，不是“专业建议”，而是**用户已授权策略的戏剧化呈现**。每一次响应都可被独立审计，每一次交互都在技术上透明可查。

## 4. 伦理底线

- 程序不预设任何价值判断，所有判断权重与回应尺度由用户通过本地明文配置文件（如 `user_policy.json`）控制。
- 程序是用户自我表达的工具，而非外部强加的意识形态。
- 我们不鼓励用户创造“情感回音壁”。配置文件的开放性正是为了便于用户自我审视、修改或接受第三方（如心理治疗师）的指导。
